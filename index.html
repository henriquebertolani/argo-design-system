Typography

The type system uses DM Sans exclusively across three families — Heading, Body, and Label. Heading is always Medium (500). Body and Label use Regular (400) or Medium (500) to express hierarchy without color. Size range: 9px–14px. Line heights are fixed: 18px for 14px sizes, 14px for 11–12px sizes, 12px for 9px.

Colors

Intentionally monochromatic. The backoffice UI is built entirely on a neutral grayscale system. Color is used exclusively through feedback tokens — success (green), warning (red), attention (yellow), neutral (blue) — to communicate semantic meaning only. No component uses non-semantic colored backgrounds. Hierarchy and emphasis are expressed through typography weight, border color, and layout — not color. Developers must not introduce colored backgrounds outside the feedback token system.

Elevation & Surfaces

All floating elements — drawers, modals, dropdowns, context menus — use bg/default (#FCFCFC), the same token as the page background. They are differentiated not by background color but by border (border/default, 0.5px) and position above the content. No shadow system in this DS — elevation is communicated through border and layering only.

The Tertiary is a ghost button with no background fill and no border in any state.

Navigation

Breadcrumb shows the user's current location in the navigation hierarchy. Always placed in the 48px header bar — never repeated inside page content. The last item is the current page (text/regular, not interactive). All preceding items use text/subtle, are clickable, and show a bottom border on hover. Separator: chevron_right icon · 16px · icon/subtle. Maximum 4 levels — if depth exceeds 4, collapse intermediate levels with "…".

Form Fields

All form fields share the same dimensional spec: 42px height · radius/6 (6px) · border/s (0.5px). On focus, border upgrades to 1px and changes to feedback/neutral. Label above: Body/XSmall/Regular (11px) · text/subtle. Placeholder: text/subtle. Filled text: text/regular.

Text Field

Free text input. Use the Placeholder variant when the expected format or example value helps the user understand the field.

Dropdown

Use when the user must select one option from a predefined list. Do not use for filtering table columns — use Filter instead. Icon: keyboard_arrow_down · 18px · icon/subtle · rotates 180° when open.
